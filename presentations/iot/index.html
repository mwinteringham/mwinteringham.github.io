Teaching with IoT

  • I have created an architecture that I use to help teach testers practise their technical testing and test design skills
  • I'm not going to give away the architecture as that's cheating, but I wanted to share some interesting points from other testers working with the device

'What! I have to learn API testing!''

  • We have to focus on IoT as an architecture not a device. There is a lot going on in the web

'I wanted to go deeper into the device!'

  • That said, there is a lot going on in the device. One student wanted to learn more about the device itself and it's potential flaws. If we're too surface level with the device we can miss issues such as security exploits.

'Why did your APIs stop working?'

  • Most of the unexpected issues were in the web architecture and not the device. It's seductive to think that the problems are in the new shiny technology and not the more standard features.

Wrapping up

  • There's no silver bullet when it comes to IoT
  • You need to understand your context to decide the best approach
  • Playing around with IoT is a great way to familiarise yourself with how to test IoT
  • And it's fun!
